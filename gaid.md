# 背景

少儿编程C++教培线上授课老师，每周都需要进行接近120个学生回访工作。语言重复，动作机械，回访内容无法与课程数据结合，人工效率极低。

# 产品定位

面向教培机构的**AI智能回访话术生成系统**，帮助授课老师将课程数据自动转化为个性化回访话术，降低机械劳动，提升回访质量与效率。

# 用户角色

| 角色 | 职责 |
|------|------|
| 授课老师 | 上传课程数据、查看/编辑/导出AI生成的话术、评估学员状态 |
| 后台管理员 | 查看各老师周度话术生成数据、管理账号、查看统计报表 |

# 核心功能模块

## 1. 课程数据上传
- 老师从现有教务系统自行导出本周课程数据，上传至本系统（支持Excel/CSV）
- 系统提供标准模板下载，方便老师对照导出所需字段
- 上传时支持字段名智能映射（老师导出字段名可能与标准表头略有差异，系统自动匹配）
- 按营期+班级维度分组解析，自动关联到当前登录老师账号
- 上传后系统校验字段完整性，异常数据标红提示
- 支持历史数据查看、重新上传与增量覆盖

**Excel标准表头字段：**

| 字段 | 说明 | AI话术权重 |
|------|------|-----------|
| 用户ID | 系统学员唯一标识 | 低 |
| 用户姓名 | 学员姓名 | 高（话术称呼） |
| 工号 | 老师工号 | 中（校验归属） |
| 老师姓名 | 授课老师姓名 | 中 |
| 营期名 | 所属营期 | 低 |
| 班级ID / 班级名 | 学员所在班级 | 低 |
| 课程序号 / 课节ID / 课程id | 课程唯一标识 | 低 |
| 课程名称 | 本次课主题 | 高（话术内容核心） |
| 学生是否到课 / 是否迟到 / 是否早退 / 是否旷课 | 出勤状态 | 高（开场提醒/关切） |
| 有效上课时长(分钟) | 实际在线时长 | 中 |
| 学生是否完课 / 是否回放完课 | 完课状态 | 中 |
| 回放观看时长（分钟） | 补看回放情况 | 中（侧面反映积极性） |
| 课中题目数 / 完成率 / 正确率 | 课堂练习表现 | 高（学习效果核心指标） |
| 课后题目数 / 完成率 / 正确率 | 课后作业表现 | 高（作业跟进核心指标） |
| 课后拓展题目数 / 完成率 / 正确率 | 拓展练习表现 | 高（拔尖/补差依据） |
| 开麦时长 / 开摄像头时长 | 课堂互动参与度 | 中（积极性侧面指标） |
| 课中获得奖杯数 | 课堂激励表现 | 中（鼓励话术素材） |
| 课节开始时间 / 课节结束时间 | 上课时间 | 低（周次归组用） |

## 2. AI话术生成引擎
- **基于课程数据自动生成**：结合出勤、完课、题目正确率、互动时长等多维数据，自动生成差异化回访话术
  - **高正确率+全勤**：鼓励巩固型，建议拓展挑战
  - **正确率低+缺勤**：关切提醒型，强调补看回放、薄弱环节
  - **完课率低**：委婉提醒家长关注学习连贯性
  - **互动少（开麦/摄像头短）**：鼓励课堂参与，建立信心
- **基于学员状态评估生成**：老师手动勾选/填写学员状态（如积极性、理解程度、需跟进事项），AI据此二次优化或独立生成话术
- 支持多种话术风格：正式/亲切/鼓励型
- 支持家长端话术（委婉表达问题）与学员端话术（直接指导）

### 话术结构规范（示例）

AI生成话术应遵循以下模块化结构，便于老师阅读与编辑：

```
【授课内容】
{{学员姓名}}本周主要学习了：
🔍 {{知识点1}}
🔍 {{知识点2}}

【课中表现】
{{学员姓名}}这周{{听课状态描述}}，课上还拿了{{奖杯数}}个奖杯，{{积极性评价}}。课后作业完成得{{作业评价}}，正确率是{{课后正确率}}%，{{作业分析}}。{{拓展练习评价}}。接下来可以{{学习建议}}。

【下节课预告】
下节课我们将学习《{{下节课名称}}》，具体知识点可以提醒孩子看看预习卡片。

【本周任务】
1. {{任务1}}
```

**数据映射关系：**

| 话术内容 | 数据来源 | 备注 |
|---------|---------|------|
| 学员姓名 | 用户姓名 | 直接引用 |
| 知识点列表 | 课程名称 | 系统维护课程名称→知识点映射表 |
| 听课状态描述 | 有效上课时长+完课状态 | 数据推断，老师可编辑 |
| 积极性评价 | 开麦时长+开摄像头时长+奖杯数 | 数据推断 |
| 奖杯数 | 课中获得奖杯数 | 直接引用 |
| 作业评价/正确率 | 课后题目完成率+正确率 | 直接引用 |
| 拓展练习评价 | 课后拓展完成率+正确率 | 直接引用 |
| 学习建议 | AI根据薄弱点/错误率生成 | 需结合课程知识体系 |
| 下节课名称 | 系统课程顺序表 | 管理员维护课程进度 |
| 本周任务 | AI根据当前课知识点生成 | 默认模板，老师可修改 |

### 多场景话术模板

系统根据学员数据自动判断场景，匹配对应话术结构：

**场景一：缺勤**

触发条件：`学生是否到课` = 否 / `是否旷课` = 是

```
【授课内容】
{{学员姓名}}本周主要学习了：
🔍 {{知识点描述}}

【课中表现】
{{学员姓名}}缺席了《{{课程名称}}》，请务必安排时间补课。

【本周任务】
1. 观看回放并完成作业
2. 预习下节课
```

数据映射：知识点描述来自课程名称映射的知识库；若缺勤则不引用正确率、奖杯数等课中数据。

**场景二：学困**

触发条件：课中正确率较低 / 课后正确率低于阈值 / 有效上课时长短 / 奖杯数为0，但课后基础作业正确率尚可

```
【授课内容】
{{学员姓名}}本周主要学习了：
【未匹配到本周知识点】

【课中表现】
{{学员姓名}}妈妈好。这周课上讲的新概念和逻辑稍微有点多，孩子听起来略微有点吃力，还在消化吸收的过程中，所以本周暂时没拿到奖杯。不过从课后作业来看，基础部分掌握得挺扎实，正确率是{{课后正确率}}%，完成得很认真。另外，这周的拓展题还没来得及做，记得提醒孩子抽空练练手感，保持做题的感觉。建议孩子可以把本周练习中不太熟的部分，自己多敲几遍代码，找找编程的节奏，这样下次课会衔接得更顺畅。

下节课我们将学习《{{下节课名称}}》，具体知识点可以提醒孩子看看预习卡片。

【本周任务】
1. 完成本周拓展训练
2. 预习下节课
```

数据映射：知识点描述优先从课程名称映射；若映射失败则显示"【未匹配到本周知识点】"，提示老师手动补充。课中表现以鼓励为主，委婉表达吃力，同时用课后基础作业正确率作为正向支撑。

| 场景 | 触发条件 | 话术特征 |
|------|---------|---------|
| 优秀型 | 全勤+高正确率+有拓展 | 鼓励巩固，建议挑战 |
| 缺勤型 | 未到课/旷课 | 补课提醒，不引用课中数据 |
| 学困型 | 课中表现弱但基础尚可 | 委婉鼓励，强调基础扎实，给出具体练习建议 |
| 互动少型 | 开麦/摄像头时长短 | 鼓励课堂参与，建立信心 |
| 完课率低 | 未回放/回放时长短 | 委婉提醒家长关注学习连贯性 |

## 3. 话术管理与编辑
- 列表展示本周所有学员及对应生成话术
- 单行快速编辑、复制
- 支持批量操作（批量导出、批量重新生成）
- 话术状态标记：已编辑、已导出、已使用

## 4. 学员状态评估
- 快捷评估表单：学习态度、知识掌握、作业质量、课堂互动、需关注事项
- 评估后可再次触发AI生成，与课程数据话术做对比或融合

## 5. 批量导出
- 导出格式：Word、Excel、PDF
- 按班级/学员分组导出
- 导出手抄本（含学员信息+话术）

## 6. 后台管理
- 老师账号管理（增删改查、重置密码）
- 周度数据看板：各老师上传数据量、生成话术条数、编辑率、导出率
- 话术质量抽查：管理员可查看老师最终使用的话术
- 系统设置：话术风格模板管理、AI生成参数配置

# 核心业务流程

```
老师登录
  → 上传本周课程数据（Excel/CSV）
  → 系统解析数据 → AI自动生成回访话术
  → 老师查看话术列表
      → 直接复制使用
      → 在线编辑后使用
      → 评估学员状态 → AI重新生成 → 编辑使用
  → 批量导出本周话术

管理员登录
  → 查看老师数据上传与话术生成统计
  → 账号与系统配置管理
```

# 数据模型（关键实体）

- **User（用户）**：id, 角色（老师/管理员）, 姓名, 账号, 密码哈希, 创建时间
- **Student（学员）**：id, 姓名, 所属老师, 班级, 家长联系方式, 创建时间
- **CourseData（课程数据）**：id, 学员id(关联Student), 用户ID, 用户姓名, 工号, 老师姓名, 营期名, 班级ID, 班级名, 课程序号, 课节ID, 课程id, 课程名称, 是否到课, 是否迟到, 是否早退, 是否旷课, 有效上课时长, 是否完课, 是否回放完课, 回放观看时长, 课中题目数, 课中完成率, 课中正确率, 课后题目数, 课后完成率, 课后正确率, 拓展题目数, 拓展完成率, 拓展正确率, 开麦时长, 开摄像头时长, 奖杯数, 课节开始时间, 课节结束时间, 数据周次, 上传批次, 上传时间
- **StatusEval（状态评估）**：id, 学员id, 评估周次, 学习态度, 知识掌握, 作业质量, 课堂互动, 需关注事项, 评估时间
- **Script（话术）**：id, 学员id, 来源类型（课程数据/状态评估）, 生成内容, 是否编辑, 最终内容, 状态, 生成时间

# 技术栈建议

- **前端**：Vue 3 / React + Element Plus / Ant Design
- **后端**：Node.js(Express/Nest) 或 Python(FastAPI/Django)
- **数据库**：PostgreSQL 或 MySQL
- **AI能力**：OpenAI API / Claude API / 国内大模型API（文心/通义）
- **文件存储**：本地或OSS（导出文件、上传模板）
- **部署**：云服务器 + Nginx，或Serverless

# 扩展性考虑

1. **话术模板库**：支持管理员预设多套话术模板，老师按场景选择
2. **话术收藏**：老师可将优质话术收藏为个人模板
3. **多科目支持**：当前为C++，后续可扩展至Python、Scratch等
4. **自动发送**：未来可对接企业微信/短信API，实现话术一键发送给家长
5. **家长反馈**：记录家长回复，形成回访闭环
6. **数据安全**：学员信息脱敏、老师数据隔离、上传文件扫描
