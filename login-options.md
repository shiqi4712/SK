# 登录方式选型分析

## 方案对比总览

| 方案 | 实现难度 | 安全性 | 用户体验 | 适用场景 |
|------|---------|--------|----------|----------|
| 账号密码 | 低 | 中 | 中 | 内部小团队，快速上线 |
| 账号密码 + 验证码 | 低 | 中高 | 中 | 需要基础防爆破保护 |
| 工号/手机号 + 密码 | 低 | 中 | 高 | 老师已有工号体系 |
| 企业微信扫码 | 中 | 高 | 高 | 机构已用企业微信 |
| 钉钉扫码 | 中 | 高 | 高 | 机构已用钉钉 |
| 短信验证码登录 | 中 | 中 | 高 | 不想记密码 |

---

## 方案一：账号密码登录（推荐MVP）

**实现方式**
- 管理员在后台为老师创建账号（用户名 + 初始密码）
- 老师首次登录后强制修改密码
- 密码 bcrypt 加密存储
- JWT Token 保持登录态

**优点**
- 实现最简单，半天可完成
- 无外部依赖，不依赖第三方平台
- 管理员完全掌控账号生命周期

**缺点**
- 老师需要记住一套新密码
- 无二次验证，安全性一般
- 离职后需管理员手动禁用账号

**适合情况**：项目初期快速上线，内部使用，人数不多（几十人）

---

## 方案二：工号 + 密码登录（推荐）

**实现方式**
- 老师用工号（Excel里已有字段）作为账号
- 初始密码为工号后6位或手机号后6位
- 首次登录强制修改密码

**优点**
- 账号体系与现有教务数据打通，老师不用记新账号
- 上传Excel时可自动校验"工号"与"登录账号"匹配
- 符合老师已有的工作习惯

**缺点**
- 需要机构有规范的工号体系
- 仍需密码管理

**适合情况**：机构已有工号体系，希望与现有数据打通

---

## 方案三：账号密码 + 图形验证码

**实现方式**
- 在方案一/二基础上，登录时增加图形验证码
- 连续输错3次密码后触发验证码
- 连续输错5次锁定账号15分钟

**优点**
- 低成本防暴力破解
- 不影响正常用户体验

**缺点**
- 略微增加实现复杂度
- 图形验证码对用户体验有小影响

**适合情况**：对外网暴露，担心被恶意登录尝试

---

## 方案四：企业微信扫码登录

**实现方式**
- 接入企业微信 OAuth2 授权登录
- 老师扫码后，企业微信返回用户身份
- 系统将企业微信 UserID 绑定到本系统老师账号

**优点**
- 老师无需记密码，扫码即登录
- 员工离职后，企业微信管理员移除人员，系统同步失效
- 机构已用企业微信发通知，体验一致

**缺点**
- 机构必须有企业微信
- 需要机构管理员在企业微信后台创建应用、配置回调地址
- 实现复杂度中等，需要处理 OAuth 流程

**适合情况**：机构已在使用企业微信进行日常沟通和管理

---

## 方案五：钉钉扫码登录

**实现方式**
- 接入钉钉 OAuth2 / 扫码登录
- 与方案四类似，只是平台换成钉钉

**优点/缺点**
- 与企业微信类似，取决于机构现有平台

**适合情况**：机构已在使用钉钉

---

## 方案六：短信验证码登录

**实现方式**
- 老师输入手机号 → 系统发送短信验证码 → 输入验证码登录
- 手机号需提前在后台绑定

**优点**
- 无需记密码，体验好
- 手机号是老师已有信息

**缺点**
- 需要接入短信平台（阿里云/腾讯云），有费用（约0.05元/条）
- 短信到达有延迟，偶尔收不到
- 需要防短信轰炸机制

**适合情况**：不想维护密码体系，且能接受短信成本

---

## 推荐组合策略

### 阶段一：MVP快速上线
**工号 + 密码**
- 用工号作为账号，降低老师记忆成本
- 管理员后台统一创建账号
- 首次登录强制修改密码

### 阶段二：体验优化（可选）
**工号 + 密码 + 记住我**
- 登录页增加"记住我"复选框，7天内免登录
- 降低老师每天登录的繁琐感

### 阶段三：深度集成（可选）
**企业微信/钉钉扫码 + 账号密码兜底**
- 优先扫码登录，一键便捷
- 同时保留账号密码登录作为备用（扫码失败、未绑定等情况）

## 数据库调整建议（如选工号登录）

```sql
-- 用户表增加工号字段
ALTER TABLE users ADD COLUMN employee_no VARCHAR(50) UNIQUE AFTER name;

-- 老师可用工号或用户名登录，建议增加登录标识字段
ALTER TABLE users ADD COLUMN login_id VARCHAR(50) UNIQUE NOT NULL COMMENT '登录标识（工号或用户名）';
```

## 决策建议

| 你的情况 | 推荐方案 |
|---------|---------|
| 要一周内上线，先跑起来 | 方案二：工号 + 密码 |
| 机构已用企业微信/钉钉 | 方案四/五：扫码登录（可保留密码兜底） |
| 老师频繁反馈记不住密码 | 方案六：短信验证码 |
| 系统暴露公网，担心安全 | 方案三：账号密码 + 验证码 + 防暴力破解 |
| 想最省事，不依赖外部平台 | 方案一：账号密码 |
